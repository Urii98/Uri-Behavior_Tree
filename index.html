The basics of
Behavior trees

Introduction

A behavior tree (BT) is a hierarchical structure of tasks and decisions that are used to model the AI(Artificial Intelligence) behavior in a game. Graphically represented as a tree, a way of structuring information hierarchically, that is widely used in Computer Science. Behavior trees are a powerful tool for game developers and other AI practitioners because they provide a flexible and intuitive way to design complex decision-making systems by breaking down the decision-making process into a series of simple, modular components(nodes). The problem that behavior trees help to solve is the challenge of creating intelligent agents that can make complex decisions in real-time, while also being responsive to player input and able to adapt to changing game conditions.

Traditional AI techniques, such as rule-based systems and finite-state machines, can quickly become unwieldy and difficult to manage as the complexity of the game increases. Behavior trees provide a more scalable and modular approach to designing AI behaviors. Additionally, behavior trees have the potential for reusability across multiple projects, which can save time and resources for game studios.

State of the art

The concept of behavior trees was first introduced in a paper by M. J. Mateas and A. Stern in 2004, and has since then evolved into one of the most common approaches for developing game AI. BTs have emerged as a popular tool Sfor modelling artificial intelligence (AI) in games since their use in the Halo 2. BTs are often used to represent the decision making of a non-player character (NPC). Many game engines, such as Unity and Unreal Engine, provide built-in support for Behavior Trees, making them easy to implement in game development.

In addition to the built-in behavior tree systems that come with game engines, there are also third-party libraries and tools that offer more advanced features and flexibility. One popular example is BehaviorTree.CPP, an open-source C++ library for creating behavior trees that is used in a wide range of applications, including video games. Another example is the Behavior Tree Editor, a visual tool for creating and editing behavior trees that can be used with various game engines and frameworks. This tool allows developers to design and test complex behavior trees more easily and quickly than writing code manually.

Although there are already libraries or software where Behavior Trees are implemented, the goal of this project was to implement a Behavior Tree from scratch that could be used in the video game projects we develop at the university. To achieve this, I gathered valuable information from sources such as books. Three particularly helpful books were:

  1. Artificial Intelligence for Games by Millington and Funge
  2. Game Coding Complete by McShaffry
  3. C++ Primer by Lippman, Lajoie, and E.Moo

Additionally, I found the following papers to be particularly informative and straightforward:

  • Marcotte, R., & Hamilton, H. J. (2017). Behavior trees for modelling artificial intelligence in games: A tutorial. The Computer Games Journal, 6, 171-184
  • Planning, M. CMSC 425: Lecture 21 Artificial Intelligence for Games: Decision Making

I also drew inspiration from previous implementations created by others, such as those found on GitHub repositories like aicore and miccol.
